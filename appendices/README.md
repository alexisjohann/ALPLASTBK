### CORE-Anforderungen (8 Pflicht)

Ein Appendix wird CORE wenn er **ALLE 8** erfüllt:

1. ✅ **Fundamentale Frage:** Beantwortet WHO, WHAT, HOW, WHEN, WHERE, AWARE, READY, oder STAGE
2. ✅ **Axiom-System:** Eigene Axiome (8-200+) mit eindeutigen Codes
3. ✅ **Literatur:** 80-150+ Referenzen, thematisch organisiert
4. ✅ **Critical Foundations:** 5-10 steelmanned Einwände + Antworten
5. ✅ **Open Issues:** 5-8 offene Fragen mit Forschungs-Roadmap
6. ✅ **Bidirektionale Integration:** Verbindung zu ALLEN anderen CORE
7. ✅ **Glossar:** 20+ Symboldefinitionen

## 📝 Wie man einen Appendix schreibt

### 1. Template verwenden

```bash
cp 00_appendix_template.tex appendix_[CODE]_[name].tex
```

### 2. 4-Teil-Struktur einhalten

```
PART 1: FRONT MATTER (Pflicht)
├── Header Block (Code, Name, Version)
├── Chapter Linkage Box
├── Abstract (max 100 Wörter)
└── Quick Reference Box

PART 2: CORE CONTENT
├── Fundamental Question
├── Core Theory + Axiome
├── Key Results
└── Integration mit anderen Appendices

PART 3: APPLICATION
├── Worked Example
└── Implications for Practice

PART 4: BACK MATTER (Pflicht)
├── Summary Box
├── Glossary of Symbols
├── Open Issues
└── References
```

### 3. Für CORE-Appendices: 8 Anforderungen prüfen

Siehe [00_appendix_template.tex](00_appendix_template.tex) für Details.
