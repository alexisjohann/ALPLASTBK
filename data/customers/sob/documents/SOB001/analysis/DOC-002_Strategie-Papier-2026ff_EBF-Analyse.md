### 5.1 Bestätigung der Strategischen Identitätslücke (H1 aus DOC-001)

```

│  DOC-001 (100-Tage-Bericht):                                           │

│   Elemente»                                                             │
│                                                                         │
│  DOC-002 (Strategie-Papier):                                            │
│  «Nicht ausreichend in die operative Steuerung integriert»              │

│                                                                         │

│                                                                         │

│  nicht nur ein AWARE-Problem (Sichtbarkeit der Strategie).              │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 5.2 Ressourcen-Constraint als Verhaltensbremse

```
4× im Dokument: «Vorhandene personelle Ressourcen definieren
                  Handlungsspielraum»

  → GB Transport:       Ressourcen = Constraint
  → GB Infrastruktur:   Ressourcen = Constraint
  → GB Mobilität:       Innovation abhängig von externer Finanzierung
  → GB Personal & Org.: Ressourcenknappheit für Transformationsprojekte

EBF-Interpretation:
  → Scarcity Mindset (Mullainathan & Shafir 2013):
    Knappheit verengt den mentalen Tunnel ("tunneling")
    → Fokus auf Dringendes, nicht Wichtiges
    → Innovation wird zum Luxusgut
    → Strategisches Denken wird verdrängt durch operativen Druck

  Implikation für SOB002:
  → Jede Intervention muss "selbstfinanzierend" sein
  → Effizienzgewinne als Argument (nicht Kosten)
  → Framing: "Was kosten wir UNS, wenn wir NICHTS tun?"
```

### 5.3 Das «Integrierte Bahnmodell» als Komplementaritäts-Asset

```
Stärke (Gesamtunternehmen):
  «Enge Verzahnung der verschiedenen GBs ermöglicht Effizienz,
   Qualität und schnelle Umsetzung.»

ABER gleichzeitig:
  → «Übergreifende Zusammenarbeit führt tlw. zu Reibungsverlusten» (Transport)
  → «Operative Anschlussfähigkeit an Transport teilweise unvollständig» (Mobilität)
  → «Fehlende strategische Verknüpfung zwischen HR und Steuerung» (Personal)

EBF-Interpretation:
  Das integrierte Bahnmodell hat POTENZIELLE Komplementarität:
    γ(Transport, Infrastruktur) > 0  (theoretisch)
    γ(Mobilität, Transport) > 0     (theoretisch)

  Aber REALISIERTE Komplementarität ist niedriger:
    γ_realisiert < γ_potenziell

  Ursache: Sequentielle Steuerung verhindert wechselseitige Wirkung.

  → SOB002 muss die Lücke γ_potenziell - γ_realisiert schliessen.
```

### 5.4 Leitsätze als «Werte-Ebene» statt «Strategie-Ebene»

```
Die 7 Leitsätze sind:
  L1: Sicherheit          → Wert (nicht messbar)
  L2: Compliance          → Pflicht (regulatorisch)
  L3: Unternehmerisch     → Haltung (nicht operationalisiert)
  L4: Nachhaltigkeit      → Wert (nicht messbar ohne KPIs)
  L5: Digitalisierung     → Mittel (nicht Ziel)
  L6: Wachstum            → Richtung (aber: "Qualität vor Quantität" = vage)
  L7: Agilität            → Eigenschaft (nicht Handlung)

EBF-Bewertung:
  → Alle 7 Leitsätze sind auf WERTE-/HALTUNGS-Ebene
  → Keiner ist auf HANDLUNGS-Ebene (messbar, zeitgebunden, verantwortlich)
  → Bestätigt DOC-001/H1: «Keine konkreten Zielsetzungen und Massnahmen»

  Strategiepyramide-Lücke:
  ┌──────────────────────┐
  │  Vision    ✅ (vorhanden)  │
  │  Mission   ✅ (SOB001 aktualisiert) │
  │  Werte     ✅ (5 Werte + 7 Leitsätze) │
  │  Ziele     ❌ (fehlen / nicht operationalisiert) │
  │  Massnahmen ❌ (fehlen) │
  │  KPIs      ❌ (fehlen in Strategie) │
  └──────────────────────┘

  → SOB002 muss die unteren 3 Ebenen füllen.
```

### 5.5 Innovationsparadox: Kein Geld für das, was Geld bringt

```
IST:
  → Innovationskraft als Stärke identifiziert (AFAS, IFIZ, AIWA)
  → «Innovationsprojekte abhängig von externer Finanzierung»
  → «Keine eigenen finanziellen Mittel für Innovation»

Paradox:
  Innovation = strategische Stärke (S5 in DOC-001)
  Innovation = finanziell nicht gesichert (Schwäche in DOC-002)

  → Innovation ist ZUFÄLLIG erfolgreich (wenn Förderung kommt),
    nicht SYSTEMATISCH verankert

EBF-Interpretation:
  → Hyperbolic Discounting: Operative Kosten (heute, sicher)
    werden übergewichtet vs. Innovationsnutzen (morgen, unsicher)
  → Present Bias β < 1: "Wir können uns Innovation jetzt nicht leisten"
  → Aber: Langfristkosten der Nicht-Innovation sind höher

  Implikation: SOB002 braucht ein "Innovations-Budget-Framing"
  → Nicht: "Was kostet Innovation?"
  → Sondern: "Was kostet UNS Nicht-Innovation in 5 Jahren?"
```

### 5.6 Regulierungsdruck als Default-Architektur

```
Regulatorisches Umfeld:
  → BAV, PBG, EBV, FABI, OR, SMS-ES, ERTMS, TSI, MODI, QMS-RPV...
  → «Weniger Differenzierungsmöglichkeiten durch regulatorische Vorgaben»
  → «Zusätzliche Nachweispflichten»

EBF-Interpretation:
  Regulierung wirkt als EXTERNE DEFAULT-ARCHITEKTUR:
  → κ_default(extern) = 0.90+ (sehr hohe Compliance-Erwartung)
  → Freiheitsgrade für SOB-eigene Defaults sinken
  → ABER: Proaktive Compliance = Wettbewerbsvorteil (L2 + Politik-Feld)

  Chance für SOB002:
  → Regulierung nicht als Bremse framen, sondern als STRUKTUR
  → «Compliance-First» kann die fehlende Handlungsleitung ERSETZEN
  → Regulatorische KPIs als Proxy für strategische Ziele nutzen
```

```

SOB002-Design:
  → BEATRIX-Dashboard als Integrationsinstrument
  → Quartals-Reviews statt Jahres-Reviews

  → GL-Workshop als Feedback-Mechanismus (nicht nur Reporting)
```

## 10. Fazit

Das Strategie-Papier ist das **analytisch stärkste Dokument** im SOB001-Ordner. Es liefert eine systematische Auslegeordnung über alle 5 Geschäftsbereiche, den regulatorischen Rahmen und das externe Umfeld. Es bestätigt und verschärft die Diagnose aus dem 100-Tage-Bericht:

> **Die SOB hat eine starke Basis (Kultur, Betrieb, Technologie), aber die Strategie ist auf der Werte-Ebene stehen geblieben.** Die Übersetzung in messbare Ziele, Massnahmen und KPIs fehlt. Gleichzeitig sind die Ressourcen knapp und die Steuerung fragmentiert.

**Die zentrale Aufgabe von SOB002:**

```
VON:  Vision → Werte → Leitsätze → ???
NACH: Vision → Mission → Werte → Leitsätze → ZIELE → MASSNAHMEN → KPIs
                                               ↑                      ↓
                                               └────── FEEDBACK ──────┘
```

Das Dokument liefert alle **Inputs** für SOB002 (Stärken, Schwächen, Chancen, Risiken, Rahmenbedingungen), aber noch keine **Outputs** (Ziele, KPIs, Interventionen). Das ist exakt der Scope von SOB002.
