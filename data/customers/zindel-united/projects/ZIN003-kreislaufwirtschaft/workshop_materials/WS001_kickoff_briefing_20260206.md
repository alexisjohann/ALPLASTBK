## Das Kernelement: Die 48h-Regel

```
┌───────────┐     ┌───────────┐     ┌───────────┐
│    🟢     │     │    🔴     │     │    🔵     │
│  TESTEN   │     │  STOPPEN  │     │ ESKALIEREN│
│           │     │           │     │           │
│ Reversibel│     │ Bewusst   │     │ Type-1    │
│ → Lern-   │     │ NEIN      │     │ → Führung │
│ Hypothese │     │ → Doku    │     │ einbezieh.│
│ → 48h     │     │ warum     │     │           │
│   Start   │     │           │     │           │
└───────────┘     └───────────┘     └───────────┘
```

**5-Fragen-Eskalationsfilter:** Mindestens 1x JA → 🔵 ESKALIEREN

1. Strategische Richtungsentscheidung?
2. Nicht begrenzbares Risiko?
3. Legitimer Zielkonflikt ohne Mandat?
4. Rahmen überschritten (Budget, Zeitplan)?
5. Verantwortung blockiert?

**5x NEIN → 🟢 TESTEN! Default = Handeln.**

## Proof of Readiness (Was der Workshop liefert)

Am Ende des Workshops sind folgende Ergebnisse dokumentiert:

- [ ] **2 Ziele** vom Team akzeptiert (Z1: Business Case, Z2: Alle mitnehmen)
- [ ] **15-20 Themen** gesammelt (Post-its)
- [ ] **3-5 Pfosten** (Lern-Felder) definiert und benannt
- [ ] **Pfosten-Owner-Matrix** vollständig
- [ ] **1 echte 🟢 TESTEN Entscheidung** getroffen (nicht Simulation!)
- [ ] **1 Lern-Hypothese** formuliert
- [ ] **1 Verantwortlicher** startet innerhalb 48h
- [ ] **Retro-Termin** in 2 Wochen steht

## Nach dem Workshop

**Innerhalb 2 Stunden:**
- Protokoll an alle senden (inkl. Fotos von Flipcharts)
- Lern-Hypothese als PDF
- Retro-Kalendereinladung

**Innerhalb 48 Stunden:**
- Check-in mit Lern-Hypothese-Owner: «Läuft der Start?»
- Check-in mit Pfosten-Ownern
- Dokumentieren: Wurde gestartet?

**Nach 2 Wochen:**
- Retro / Lern-Loop durchführen
- Lern-Hypothese auswerten
- Nächste 🟢 TESTEN Entscheidungen
