## 1. WARUM SKILL-BASED? DAS PROBLEM IN DREI SÄTZEN

Heute werden Mitarbeitende nach Stellenbeschreibungen eingestellt, nach Dienstjahren befördert und nach Verfügbarkeit eingesetzt. Das führt zu drei messbaren Problemen:

```
┌────────────────────────────────────────────────────────────────────┐
│                                                                    │
│  PROBLEM 1: SKILL-BLINDHEIT                                       │
│  → 45% der Mitarbeitenden haben Skills, die niemand kennt         │
│  → Ergebnis: Falsche Besetzung, verpasste Chancen                 │
│                                                                    │
│  PROBLEM 2: STARRE KARRIEREPFADE                                  │
│  → Beförderung = Dienstjahre + Verfügbarkeit                     │
│  → Ergebnis: Demotivation der Leistungsträger:innen               │
│                                                                    │
│  PROBLEM 3: MANGELNDE FLEXIBILITÄT                                │
│  → Im Schnitt beherrscht jede Person 1.2 Rollen                  │
│  → Ergebnis: Jeder Krankenstand wird zur Krise                    │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

**Das Ziel:** Von 1.2 Rollen pro Person auf 2.5 — bei gleichzeitig besserer Qualität, fairerer Bewertung und höherer Mitarbeiterzufriedenheit.

## 2. DAS MODULARE SKILL-FRAMEWORK: 5 SKILL-EBENEN

### 2.1 Welche Skills braucht die Organisation?

Die Forschung (Autor 2003, Heckman 2026) zeigt: Es gibt nicht «eine Liste von Skills», sondern **5 fundamental verschiedene Skill-Typen**, die auf unterschiedlichen Organisationsebenen unterschiedlich wichtig sind.

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  DIE 5 SKILL-TYPEN (nach David Autor, MIT)                              │
│                                                                         │
│  ┌──────────┐  Wiederkehrende, regelbasierte Tätigkeiten                │
│  │ ROUTINE  │  Beispiele: Kassiervorgänge, Bestandserfassung,           │
│  │          │  Wareneingangskontrolle, Preisauszeichnung                │
│  └──────────┘                                                           │
│                                                                         │
│  ┌──────────┐  Körperliche, umgebungsabhängige Tätigkeiten              │
│  │ MANUELL  │  Beispiele: Regalbestückung, Frischetheke,                │
│  │          │  Warenverräumung, Ladenbau                                │
│  └──────────┘                                                           │
│                                                                         │
│  ┌──────────────┐  Kommunikation, Zusammenarbeit, Kundenarbeit         │
│  │ INTERAKTIV   │  Beispiele: Kundenberatung, Teamkoordination,        │
│  │              │  Reklamationsbearbeitung, Einarbeitung Neuer          │
│  └──────────────┘                                                       │
│                                                                         │
│  ┌──────────┐  Analytisches Denken, Problemlösung, Planung             │
│  │ ABSTRAKT │  Beispiele: Sortimentsanalyse, Personalplanung,          │
│  │          │  Budgetsteuerung, Projektmanagement                      │
│  └──────────┘                                                           │
│                                                                         │
│  ┌────────────┐  Führung, Motivation, strategische Steuerung           │
│  │ LEADERSHIP │  Beispiele: Teamführung, Changemanagement,             │
│  │            │  Feedbackgespräche, strategische Entscheidungen         │
│  └────────────┘                                                         │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 Welche Ebene braucht welche Skills?

Das Framework ist **modular**: Je nach Rolle werden unterschiedliche Skill-Typen betont. Was nicht relevant ist, wird weggelassen.

```
                        ROUTINE  MANUELL  INTERAKTIV  ABSTRAKT  LEADERSHIP
                        ─────────────────────────────────────────────────
  FRONTLINE             ████████ ████████ ██████      ██        ░░
  (Kassierer:in,        Hoch     Hoch     Mittel      Gering    —
   Regalservice)

  FACHKRAFT             ██████   ██████   ████████    ████      ░░
  (Frischetheke,        Mittel   Mittel   Hoch        Mittel    —
   Backshop)

  TEAMLEITUNG           ████     ████     ████████    ██████    ██████
  (Abteilungsleitung)   Mittel   Mittel   Hoch        Hoch      Hoch

  FILIALLEITUNG         ██       ██       ████████    ████████  ████████
                        Gering   Gering   Hoch        Sehr hoch Sehr hoch

  ZENTRALE              ░░       ░░       ██████      ████████  ██████
  (Einkauf, HR,         —        —        Hoch        Sehr hoch Hoch
   Logistik)
```

**Das Modulare Prinzip:** Für eine Frontline-Kraft werden LEADERSHIP-Skills nicht erfasst — das spart Zeit und verhindert Frustration. Für die Filialleitung werden ROUTINE-Skills nicht mehr bewertet — der Fokus liegt auf dem, was zählt.

## 3. SKILL-ASSESSMENT: WIE MESSEN WIR SKILLS?

### 3.1 Das 5-Stufen-Kompetenzmodell (nach Dreyfus)

Für jeden Skill wird ein klarer Reifegrad gemessen — nicht subjektiv, sondern nach beobachtbarem Verhalten:

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  STUFE 1: ANFÄNGER:IN                                                   │
│  ─────────────────────                                                  │
│  «Braucht Anleitung»                                                    │
│  → Kennt die Regeln, kann unter Aufsicht arbeiten                      │
│  → Typisch: Erste 3 Monate nach Einarbeitung                          │
│  → Messbar: Braucht >2× Nachfragen pro Schicht                        │
│                                                                         │
│  STUFE 2: FORTGESCHRITTEN                                               │
│  ─────────────────────                                                  │
│  «Arbeitet selbstständig»                                               │
│  → Erledigt Standardsituationen allein, fragt bei Ausnahmen           │
│  → Typisch: Nach 6-12 Monaten Praxis                                  │
│  → Messbar: Standardprozesse ohne Fehler, Ausnahmen mit Rückfrage     │
│                                                                         │
│  STUFE 3: KOMPETENT                                                     │
│  ─────────────────────                                                  │
│  «Löst Probleme eigenständig»                                          │
│  → Erkennt Prioritäten, trifft eigene Entscheidungen                  │
│  → Typisch: Nach 2-3 Jahren Erfahrung                                  │
│  → Messbar: Eigenständige Problemlösung, gibt Feedback an Neue        │
│                                                                         │
│  STUFE 4: ERFAHREN                                                      │
│  ─────────────────────                                                  │
│  «Überblickt das Ganze»                                                 │
│  → Erkennt Muster, antizipiert Probleme, coacht andere                │
│  → Typisch: Ab 4-5 Jahren mit bewusster Weiterentwicklung             │
│  → Messbar: Wird von Kolleg:innen als Ansprechperson gesucht          │
│                                                                         │
│  STUFE 5: EXPERT:IN                                                     │
│  ─────────────────────                                                  │
│  «Handelt intuitiv richtig»                                             │
│  → Tiefes Verständnis, innoviert, entwickelt Prozesse weiter           │
│  → Typisch: Ab 7+ Jahren mit kontinuierlichem Lernen                  │
│  → Messbar: Entwickelt neue Lösungen, bildet Trainer:innen aus         │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

**Wichtig:** Die Stufen sind an *Verhalten* gebunden, nicht an Dienstjahre. Eine engagierte Person kann Stufe 3 in 18 Monaten erreichen, während eine andere nach 5 Jahren auf Stufe 2 bleibt — und beides ist in Ordnung.

### 3.2 Das Assessment-System: Drei Perspektiven

Keine einzelne Bewertung ist fair. Deshalb nutzt das System **drei Perspektiven**, die zu einem validen Gesamtbild führen:

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│                          SKILL-PROFIL                                    │
│                        ┌───────────┐                                    │
│                        │ Gesamtbild │                                   │
│                        └─────┬─────┘                                    │
│                 ┌────────────┼────────────┐                             │
│                 │            │            │                              │
│          ┌──────┴──────┐ ┌──┴───┐ ┌──────┴──────┐                      │
│          │ SELBST-     │ │ PEER │ │ FÜHRUNGS-   │                       │
│          │ EINSCHÄTZUNG│ │      │ │ BEWERTUNG   │                       │
│          └─────────────┘ └──────┘ └─────────────┘                       │
│                                                                         │
│  SELBST (30% Gewicht)                                                   │
│  → Mitarbeiter:in bewertet eigene Skills auf Stufe 1-5                 │
│  → Fördert Reflexion und Eigenverantwortung                            │
│  → Typischer Bias: Überschätzung um +0.3 Stufen (korrigierbar)        │
│                                                                         │
│  PEER (30% Gewicht)                                                     │
│  → 2-3 Kolleg:innen bewerten gegenseitig                              │
│  → Erfasst Skills, die nur im Alltag sichtbar sind                    │
│  → Stärke: Interaktive und manuelle Skills besonders valide            │
│                                                                         │
│  FÜHRUNGSKRAFT (40% Gewicht)                                            │
│  → Direkte Vorgesetzte bewertet mit strukturiertem Bogen               │
│  → Verantwortlich für Kalibrierung und Entwicklungsgespräch            │
│  → Stärke: Abstrakte und Leadership-Skills besonders valide            │
│                                                                         │
│  ERGEBNIS: Gewichteter Score pro Skill                                  │
│  Score(Skill) = 0.30 × Selbst + 0.30 × Peer + 0.40 × Führung         │
│                                                                         │
│  KALIBRIERUNG (quartalsweise):                                          │
│  → Filialleiter:innen vergleichen Bewertungen im Kalibrierungstag     │
│  → Verhindert systematische Über-/Unterbewertung                       │
│  → BetrVG-konform: BR hat Mitbestimmung beim Verfahren (§94)           │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 4. FÜHRUNGSKRÄFTE-BEWERTUNG: DAS LEADERSHIP-MODUL

### 4.1 Warum ein eigenes Modul für Führungskräfte?

Führungskräfte haben einen **überproportionalen Hebel** auf die gesamte Organisation:

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  MULTIPLIKATOR-EFFEKT DER FÜHRUNGSKRAFT                                 │
│                                                                         │
│  Eine Filialleitung mit Skill-Level 4 in «Teamführung»:                │
│  → Steigert Lerngeschwindigkeit des Teams um +8%                       │
│  → Erhöht psychologische Sicherheit um +15%                            │
│  → Reduziert Fluktuation um -12%                                       │
│  → Effekt auf 25-40 Mitarbeitende gleichzeitig                         │
│                                                                         │
│  Eine Filialleitung mit Skill-Level 2 in «Teamführung»:                │
│  → Bremst Lerngeschwindigkeit um -5%                                   │
│  → Senkt psychologische Sicherheit um -10%                             │
│  → Erhöht Fluktuation um +8%                                           │
│  → Negativer Effekt auf 25-40 Mitarbeitende gleichzeitig               │
│                                                                         │
│  → Die Differenz zwischen Level 2 und Level 4 bei einer                │
│    Filialleitung ist GRÖSSER als 10 Einzeltrainings                    │
│    für Frontline-Kräfte.                                                │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 4.2 Die 6 Leadership-Dimensionen

Führungskräfte werden auf **6 Dimensionen** bewertet, die über die Standard-5-Skill-Typen hinausgehen:

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  LEADERSHIP-DIMENSION          WAS WIRD GEMESSEN?                       │
│  ───────────────────────────────────────────────────────────────────    │
│                                                                         │
│  L1  SKILL-ENTWICKLUNG         Kann die FK das Team skill-basiert      │
│      (Coaching & Enabling)     entwickeln? Führt sie regelmässige      │
│                                Entwicklungsgespräche? Erkennt sie      │
│                                Potenzial?                               │
│                                                                         │
│  L2  FEEDBACK-QUALITÄT         Gibt die FK konstruktives, spezifisches │
│      (Assessment & Review)     Feedback? Nutzt sie den Assessment-     │
│                                Bogen korrekt? Kalibriert sie fair?     │
│                                                                         │
│  L3  EINSATZSTEUERUNG          Setzt die FK Mitarbeitende              │
│      (Matching & Deployment)   skill-basiert ein? Nutzt sie den        │
│                                Matching-Score? Fördert sie              │
│                                Rollenvielfalt?                          │
│                                                                         │
│  L4  VERÄNDERUNGSFÜHRUNG       Unterstützt die FK aktiv die SBO-       │
│      (Change Leadership)       Transformation? Kommuniziert sie das    │

## 5. SKILL-BASIERTE BESETZUNG: VOM LEBENSLAUF ZUM MATCH-SCORE

### 5.1 Das heutige System vs. das zukünftige System

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  HEUTE: LEBENSLAUF-BASIERT           MORGEN: SKILL-BASIERT              │
│  ──────────────────────────          ─────────────────────              │
│                                                                         │
│  Wer wird befördert?                 Wer wird befördert?               │
│  → Wer am längsten da ist            → Wer die passenden Skills hat    │
│  → Wer die FK kennt                  → Wer den höchsten Match hat      │
│  → Wer «dran» ist                    → Wer Potenzial zeigt (Lernkurve) │
│                                                                         │
│  Wer macht den Job?                  Wer macht den Job?                │
│  → Wer auf dem Dienstplan steht      → Wer den besten Match-Score hat  │
│  → Wer gerade verfügbar ist          → Wer am meisten davon lernt      │
│  → Wer «das schon immer macht»       → Wer die Lücke am besten füllt  │
│                                                                         │
│  Was passiert bei Engpass?           Was passiert bei Engpass?          │
│  → Hektische Telefonate              → System zeigt 3 beste Optionen   │
│  → «Wer kann einspringen?»           → Match-Score + Verfügbarkeit    │
│  → Oft: Falsche Person, richtig      → Immer: Richtige Skills,        │
│    motiviert aber überfordert            richtige Rolle                  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 5.2 Der Match-Score: So funktioniert er

Für jede Rolle gibt es ein **Soll-Profil** (welche Skills auf welchem Level werden gebraucht?). Jede Mitarbeiter:in hat ein **Ist-Profil** (welche Skills auf welchem Level sind vorhanden?). Der Match-Score vergleicht beides:

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  MATCH-SCORE = Durchschnitt über alle relevanten Skills:                │
│                                                                         │
│  Match(Person, Rolle) = Ø [ min(Skill_ist / Skill_soll, 1) ]          │
│                                                                         │
│  BEISPIEL: Anna M. → Rolle «Fachtrainerin Frische»                    │
│                                                                         │
│  Skill              Soll   Ist   Score                                  │
│  ─────────────────────────────────────                                  │
│  Frischetheke         5      5    1.00  ✅                              │
│  Kundenberatung       4      5    1.00  ✅                              │
│  Einarbeitung Neuer   4      3    0.75  ⚠️                              │
│  Reklamation          3      4    1.00  ✅                              │
│  Wareneingangsk.      3      4    1.00  ✅                              │
│  ─────────────────────────────────────                                  │
│  MATCH-SCORE                       0.95 → 95% Match                    │
│                                                                         │
│  INTERPRETATION:                                                        │
│  ≥ 0.90  «Bereit»          → Kann sofort eingesetzt werden             │
│  0.80    «Fast bereit»     → 1-2 Skills noch entwickeln               │
│  0.60    «Entwicklungspfad»→ Gezieltes Training (3-6 Monate)          │
│  < 0.60  «Nicht passend»   → Andere Rolle empfohlen                    │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 5.3 Drei Anwendungsfälle für den Match-Score

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  ANWENDUNG 1: INTERNE BESETZUNG                                        │
│  ─────────────────────────────────                                      │
│  Offene Stelle «Abteilungsleitung Frische»:                            │

### A. Wissenschaftliche Grundlage

Dieses Framework basiert auf 7 wissenschaftlich validierten Theorien:

| Theorie | Forscher:in | Kern-Erkenntnis |
|---------|-------------|-----------------|
| Skill Formation Technology | Cunha & Heckman (2007) | Skills bauen aufeinander auf — frühe Investition lohnt sich exponentiell |
| Dynamic Complementarity | Heckman et al. (2025) | Vorhandene Skills beschleunigen das Lernen neuer Skills |
| Task-Based Approach | David Autor (2003) | 5 fundamental verschiedene Skill-Typen, unterschiedlich automatisierbar |
| Identity Economics | Akerlof & Kranton (2000) | Identität bestimmt Motivation: «Bin ich Kassierer:in oder Fachkraft?» |
| Supermodularity | Milgrom & Roberts (1990) | Komplementäre Investments verstärken sich gegenseitig |
| Dreyfus Model | Dreyfus & Dreyfus (1980) | 5 Stufen von Anfänger bis Expert:in — beobachtbar und messbar |
| Crowding-Out | Bénabou & Tirole (2003) | Monetäre Anreize können intrinsische Motivation zerstören |

### C. Glossar

| Begriff | Erklärung |
|---------|-----------|

| **Match-Score** | Prozentuale Übereinstimmung Person ↔ Rolle (0-100%) |
| **Flex-Score** | Durchschnittliche Anzahl Rollen pro Person mit Match ≥ 80% |

| **BetrVG** | Betriebsverfassungsgesetz — regelt Mitbestimmung |
