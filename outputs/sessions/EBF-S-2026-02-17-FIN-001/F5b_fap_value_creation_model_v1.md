## 0. Das Problem: Warum braucht F5 ein Value Model?

F5 beschreibt 4 Werthebel (Diagnose, Simulation, Design, Transformation) — aber das ist eine **Fähigkeits-Liste**, kein **Wertschöpfungs-Modell**.

```
┌─────────────────────────────────────────────────────────────────────────┐
│  WAS F5 HAT:                        WAS F5 NICHT HAT:                  │
│  ──────────────────────────────────  ───────────────────────────────── │
│  «Wir können Diagnose»              «Warum ist Diagnose wertvoll?»    │
│  «Wir können Simulation»            «Was passiert ohne Simulation?»   │
│  «70% [D] bei Paket 1»              «Wieviel CHF ist das wert?»       │
│                                                                         │
│  = Liste der FAP-FÄHIGKEITEN        = KAUSALE LOGIKKETTE               │
│    (angebotsgetrieben)                (nachfragegetrieben)              │
│                                                                         │
│  Für die Offerte brauchen wir beides.                                  │
└─────────────────────────────────────────────────────────────────────────┘
```

## 1. Das Modell: Behavioral Leaks × FAP Closure

### 1.1 Kernidee

```
┌─────────────────────────────────────────────────────────────────────────┐
│  FAP VALUE CREATION MODEL                                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  THESE:                                                                 │
│  Marketing hat «Behavioral Leaks» — Stellen, an denen menschliche      │
│  Entscheidungsverzerrungen zu suboptimalen Ergebnissen führen.          │
│  Diese Leaks existieren in JEDER der 5 Marketing-Funktionen.           │
│                                                                         │
│  MECHANISMUS:                                                           │
│  Behavioral Science identifiziert und schliesst diese Leaks.           │
│  Nicht durch mehr Budget oder bessere Technologie — sondern durch      │
│  BESSERES VERSTÄNDNIS von Entscheidungsverhalten.                      │
│                                                                         │
│  FORMEL:                                                                │
│                                                                         │
│  V_FAP = Σᵢ (Leak_i × Closure_i × Scale_i)                             │
│                                                                         │
│  Wobei:                                                                 │
│  ├── Leak_i    = Grösse des Behavioral Leak in Funktion i              │
│  ├── Closure_i = FAP-Fähigkeit, diesen Leak zu schliessen             │
│  └── Scale_i   = Anzahl betroffener Kampagnen/Touchpoints              │
│                                                                         │
│  PROVENANCE DER FORMEL:                                                 │
│  ├── Dass Behavioral Leaks existieren:       [D] — Literatur           │
│  ├── Dass sie in UBS-Marketing existieren:   [A] — Meeting-Feedback   │
│  ├── Dass FAP sie schliessen kann:           [D/A] — Methodik da      │
│  └── Die GRÖSSE der Leaks bei UBS:           [H] — nicht gemessen     │
│                                                                         │
│  → Paket 1 MISST die Leaks. Erst dann wird V_FAP quantifizierbar.     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 Leak L1: Segmentierungs-Leak (F1 Verstehen)

```
┌─────────────────────────────────────────────────────────────────────────┐
│  L1: SEGMENTIERUNGS-LEAK                                                │
│  Funktion: F1 VERSTEHEN                                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  WAS PASSIERT:                                                          │
│  Marketing segmentiert nach Demografie (Alter, Einkommen, Region).     │
│  Innerhalb dieser Segmente verhält sich aber nicht jede:r gleich.      │
│                                                                         │
│  BEISPIEL [A — basierend auf Meeting-Feedback]:                         │
│  Segment «affluent 55-65, Deutschschweiz» enthält:                     │
│  ├── Typ A: Sicherheitsorientiert (λ hoch, Status-Quo-Bias)           │
│  │   → Reagiert auf: «Bewahren Sie Ihren Lebensstandard»              │
│  ├── Typ B: Wachstumsorientiert (λ tief, Gain-Seeking)                │
│  │   → Reagiert auf: «Maximieren Sie Ihre Rendite»                    │
│  └── Typ C: Delegierer (Entscheidungslast hoch, will Empfehlung)      │

│  Dieselbe Kampagne an alle drei → 2 von 3 falsch angesprochen.        │
│                                                                         │
│  GRÖSSE DES LEAKS [A/H]:                                               │
│  Typischerweise reagieren 15-30% einer demografischen Zielgruppe       │

│  das auf 40-60% steigern [A — Literatur-Range, nicht UBS-spezifisch]. │
│                                                                         │
│  → Potenzielle Verbesserung: +67-100% Targeting-Genauigkeit [A/H]     │
│                                                                         │
│  WIE FAP DEN LEAK SCHLIESST:                                           │
│  ├── Behavioral Segmentation via 10C Framework [D — Methodik da]       │
│  ├── Entscheidungstypen identifizieren (Loss-Averse, Gain-Seeking,    │
│  │   Delegators, Status-Quo) [D — EBF-Parameter]                       │
│  └── Kampagnen-Matching: Botschaft × Typ [D/A]                         │
│                                                                         │
│  FAP CLOSURE RATE: 60-70% [A]                                          │
│  (Wir können den Leak deutlich reduzieren, aber nicht eliminieren —    │
│  Datenqualität bei UBS ist eine Constraint)                            │
│                                                                         │
│  REFERENZ:                                                              │
│  ├── EBF-Theorie: MS-IB-001 (Akerlof & Kranton, Identity Economics)   │
│  ├── EBF-Cases: CAS-445 (Behavioral Segmentation Finance)              │
│  └── Literatur: Behavioral segmentation outperforms demographic        │
│      in financial services (Thaler & Sunstein 2008, adapted) [D]      │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────────┐
```

```
│  Funktion: F3 KOMMUNIZIEREN                                             │

│                                                                         │
│  WAS PASSIERT:                                                          │

│                                                                         │
│  Besserer Zeitpunkt: März/April (Steuererklärung = Trigger für        │

│  ├── Timing-Effekte: ±10-25% Conversion je nach Timing [A — Lit.]    │

│  ├── Life-Event-Mapping: Wann sind UBS-Kund:innen empfänglich? [A]    │
```

### 2.5 Leak L4: Friktions-Leak (F4 Konvertieren)

```
┌─────────────────────────────────────────────────────────────────────────┐
│  L4: FRIKTIONS-LEAK                                                     │
│  Funktion: F4 KONVERTIEREN                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  WAS PASSIERT:                                                          │
│  Kund:innen sind interessiert, aber die Conversion-Journey hat          │
│  Friktionen: zu viele Schritte, falsche Defaults, fehlende             │
│  Orientierung, Entscheidungsüberlastung.                               │
│                                                                         │
│  BEISPIEL [A — basierend auf Branchenstandard]:                         │
│  Hauptbank-Wechsel-Journey bei einer Grossbank:                        │
│  ├── 1. Landingpage mit 6 Optionen (Choice Overload)                  │
│  ├── 2. Formular mit 23 Feldern (Friction)                            │
│  ├── 3. Kein Default gewählt (Entscheidungslast beim Kunden)          │
│  ├── 4. «Sind Sie sicher?»-Dialog (Uncertainty Amplification)         │
│  └── 5. Bestätigung erst nach 3-5 Werktagen (Delayed Reward)         │
│                                                                         │
│  Jeder dieser Schritte ist ein Behavioral Leak:                        │
│  ├── Choice Overload: >3 Optionen → Entscheidungsparalyse [D]        │
│  ├── Friction: Jedes zusätzliche Feld → -3-5% Completion [D/W]       │
│  ├── Missing Default: Ohne Empfehlung → 20-30% weniger Abschluss [D] │
│  ├── Uncertainty: «Sind Sie sicher?» → 10-15% brechen ab [D]         │
│  └── Delayed Reward: Sofortige Bestätigung → +20% Zufriedenheit [D]  │
│                                                                         │
│  GRÖSSE DES LEAKS [D/A]:                                               │
│  ├── Typische Onboarding-Completion-Rate Grossbanken: 30-45% [A/W]    │
│  ├── Behavioral-optimierte Completion-Rate: 55-70% [A — Lit.]        │
│  └── Key Club spezifisch: Hauptbank-Wechsel ist einer der             │
│      friktionsreichsten Prozesse im Retail Banking [A]                │
│                                                                         │
│  → Potenzielle Verbesserung: +50-80% Completion-Rate [A/H]            │
│    (von z.B. 35% auf 55-60%)                                          │
│                                                                         │
│  WIE FAP DEN LEAK SCHLIESST:                                           │
│  ├── Friction Audit: Jeden Schritt jeder Journey prüfen [D]           │
│  ├── Default-Optimierung: Intelligente Vorauswahl [D]                  │
│  ├── Progressive Disclosure: Weniger Felder, mehr Schritte [D]        │
│  ├── Commitment Devices: Kleine Schritte → grosse Entscheidung [D]    │
│  └── BEATRIX Journey-Simulation: Vor Redesign simulieren [D/A]        │
│                                                                         │
│  FAP CLOSURE RATE: 60-70% [D/A]                                        │
│  (Friction-Reduktion ist gut erforscht und gut umsetzbar)             │
│                                                                         │
│  REFERENZ:                                                              │
│  ├── EBF-Theorie: MS-NU-001 (Choice Overload), MS-NU-002 (Defaults)  │
│  ├── EBF-Cases: CAS-234 (Onboarding Optimization)                     │
│  ├── EBF-Parameter: PAR-BEH-003 (Default Effect Strength)             │
│  └── Literatur: Sunk cost, friction, defaults (Thaler 2015) [D]      │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.6 Leak L5: Mess-Leak (F5 Lernen)

```
┌─────────────────────────────────────────────────────────────────────────┐
│  L5: MESS-LEAK                                                          │
│  Funktion: F5 LERNEN                                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  WAS PASSIERT:                                                          │
│  Marketing misst die falschen Dinge. CTR, CPC, Impressions messen     │
│  AUFMERKSAMKEIT — nicht ENTSCHEIDUNGEN. Eine Kampagne kann hohen       │
│  CTR und niedrige Conversion haben, weil Klick ≠ Handlung.            │
│                                                                         │
│  BEISPIEL [A]:                                                          │
│  ├── Kampagne A: CTR 4.2%, aber nur 0.3% Conversion → schlecht       │
│  │   (aufmerksamkeitsstark, aber nicht handlungsauslösend)            │
│  └── Kampagne B: CTR 1.8%, aber 2.1% Conversion → gut                │
│      (weniger Klicks, aber die Richtigen)                              │
│                                                                         │
│  Wer nur CTR misst, skaliert Kampagne A. Das ist der Mess-Leak.       │
│                                                                         │
│  DAS TIEFERE PROBLEM [A/H]:                                            │
│  ├── Correlation ≠ Causation: «Kampagne X lief gleichzeitig mit       │
│  │   Sales-Peak» → War es die Kampagne oder der Markt?                │
│  ├── Attribution: Multi-Touch Journeys machen es unmöglich,           │
│  │   einzelne Kampagnen kausal zu bewerten                             │
│  └── Optimization Loop: Falsche Metrik → Falsche Optimierung          │
│      → Sieht besser aus (CTR ↑) → Wirkt nicht besser (Conversion =)  │
│                                                                         │
│  GRÖSSE DES LEAKS [A]:                                                  │
│  Schwer zu quantifizieren, weil der Leak im Messsystem selbst sitzt.  │
│  Indirekt: Unternehmen die auf Behavioral Outcomes umstellen,         │
│  allokieren Marketing-Budget 15-25% effizienter [A — Literatur-Range] │
│                                                                         │
│  → Potenzielle Verbesserung: +15-25% Budget-Effizienz [A/H]           │
│                                                                         │
│  WIE FAP DEN LEAK SCHLIESST:                                           │
│  ├── Behavioral Outcome Metrics definieren [D/A]                       │
│  │   (Nicht: «Wie viele klickten?» Sondern: «Wie viele handelten?»)  │
│  ├── Causal Attribution Framework [A/H]                                │
│  │   (Schwieriger — erfordert Kontrollgruppen oder Quasi-Experimente) │
│  └── BEATRIX Counterfactual Analysis [H]                               │
│      (Was wäre ohne die Kampagne passiert? — Simulationsbasiert)      │
│                                                                         │
│  FAP CLOSURE RATE: 30-40% [A/H]                                        │
│  (Am schwierigsten zu schliessen — erfordert Datenzugang,             │
│  Organisationsänderung, und teilweise neue Systeme)                    │
│                                                                         │
│  REFERENZ:                                                              │
│  ├── EBF-Theorie: Measurement als Ψ-Dimension (Kontext beeinflusst   │
│  │   was gemessen wird)                                                │
│  └── Literatur: Behavioral outcome metrics in marketing [A]           │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 3. Die Value Equation: Zusammenführung

```
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  V_FAP = Σᵢ (Leak_i × Closure_i × Scale_i)                             │
│                                                                         │
│  LEAK                          CLOSURE   SCALE          PROVENANCE      │
│  ─────────────────────────     ───────   ──────         ──────────      │
```
